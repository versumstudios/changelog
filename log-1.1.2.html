RPC Node - Introduction

Versum allows you to choose which RPC node you want to use to interact with the platform, your wallet and the tezos blockchain. This is a very useful feature, particularly during periods where the public nodes are extremely busy.

When you first login to versum an RPC Node will automatically be assigned to you, so as to spread the users evenly through all the available public nodes and avoid adding too much load to any one node.

Over the last couple of weeks we have received reports of users having problems interacting with their wallets. After some investigation we found the cause of the issue: teznode.letzbake.com has been deprecated, and thus any user on that node would experience issues.

The fix was quite easy, you simply need to go to your versum settings [Settings/Global] and change to another node.

However in an effort to simplify things, and to give more options to moderate and advanced users, we have refactored our code and improved the user experience, by adding automatic node reassignment, and giving users the ability to use a private node.

RPC Node (Auto)

From this update onwards every user will have this option selected by default. If your node is instable or unreachable versum will automatically change nodes for you behind the scenes.

RPC Node (Manual)

Manual mode allows you to specify one of the publicly available RPC nodes.

RPC Node (Custom)

If you are an advanced user running your own node, you can specify the url here, and Versum will use this node.
